# FIAR-Saúde — Submissão de Projetos e Templates de Documentação

Este repositório disponibiliza os **templates de documentação utilizados no processo de auditoria do framework FIAR-Saúde**.

Os documentos devem ser preenchidos pelas equipes dos projetos participantes para registrar evidências relacionadas às dimensões de **Inteligência Artificial Responsável**.

**Importante**

Ao clicar nos links abaixo, será criada **uma cópia do documento no seu Google Drive**, que poderá ser editada pela equipe do projeto.

Após o preenchimento, compartilhe o link do documento com a equipe de auditoria.

---


## 1. Documentação Inicial do Projeto

Antes do início da auditoria, cada projeto deve preencher o **Project Dossier**, que registra:

- informações gerais do projeto
- escopo do modelo
- equipe responsável
- evidências iniciais relacionadas às dimensões de IA responsável
- links para artefatos do projeto

Criar uma cópia do template: [FIAR Documentação Inicial do Projeto](https://docs.google.com/document/d/1413BabalmWr3lH9xqPT37aNQ3yF7m-PL-qYjOsisA7I/copy)

Este documento deve ser preenchido **antes do primeiro ciclo de auditoria**.

---

## 2. Relatórios por Dimensão

Após a análise inicial, a equipe de auditoria poderá solicitar relatórios adicionais para cada dimensão do framework.

Esses relatórios documentam **evidências mais detalhadas sobre o sistema de IA**.

---

### Justiça (Fairness)

Documenta a avaliação de possíveis **disparidades entre grupos relevantes**.

Exemplos de conteúdo:

- identificação de subgrupos
- métricas de fairness
- análise de disparidades
- interpretação dos resultados

Criar uma cópia do template: [Fairness Audit Report](https://docs.google.com/document/d/SEU_LINK/copy)

---

### Explicabilidade (Explainability)

Documenta os métodos utilizados para **interpretar o comportamento do modelo**.

Exemplos de conteúdo:

- descrição do modelo
- métodos de explicabilidade
- visualizações ou exemplos de explicação
- limitações das explicações

Criar uma cópia do template: [Explainability Audit Report](https://docs.google.com/document/d/SEU_LINK/copy)

---

### Privacidade

Documenta as práticas de **proteção e governança de dados** utilizadas pelo projeto.

Exemplos de conteúdo:

- descrição de dados sensíveis
- procedimentos de anonimização
- controle de acesso
- políticas de retenção de dados

Criar uma cópia do template: [Privacy Audit Report](https://docs.google.com/document/d/SEU_LINK/copy)

---

## Governança

Documenta os processos institucionais relacionados ao desenvolvimento e supervisão do sistema.

Exemplos de conteúdo:

- responsabilidades institucionais
- protocolos de decisão
- monitoramento do sistema
- conformidade com regulamentos

Criar uma cópia do template: [Governance Audit Report](https://docs.google.com/document/d/SEU_LINK/copy)

---

## 3. Processo de Auditoria

O processo de avaliação no FIAR-Saúde segue **ciclos iterativos de auditoria**.

### Etapas principais

1. Submissão da documentação inicial do projeto  
2. Auditoria preliminar pela equipe FIAR  
3. Identificação de lacunas de evidência  
4. Solicitação de relatórios por dimensão  
5. Submissão das evidências pelos projetos  
6. Avaliação técnica pela equipe de auditoria  
7. Consolidação das recomendações

Cada ciclo gera **recomendações para melhoria da documentação e das práticas do projeto**.

---

## 4. Estrutura de Artefatos Esperados

Os projetos podem ser solicitados a fornecer os seguintes artefatos:

- `project_dossier`
- `model_card`
- `data_card`
- `metrics_report`
- `fairness_report`
- `explainability_report`
- `privacy_report`
- `governance_report`
- scripts ou notebooks relevantes

Esses artefatos ajudam a garantir **transparência, rastreabilidade e auditabilidade do sistema de IA**.
